# ADR-007: 教材音声とキューをデバイスローカルに保存する

## ステータス

採用 (2026-07)

## コンテキスト

シャドーイング学習のため、教材テキストに対応する MP3 音声の再生・速度調整 (0.6〜1.0 倍)・再生位置に同期した自動ページめくりを追加した。自動ページめくりには「各行が音声の何秒から始まるか」のタイミングデータ (キュー) が必要になる。

音声とキューの置き場所として、次の選択肢があった。

1. リポジトリにコミットし GitHub Pages から配信する (`public/audio/` + 教材 JSON にキーを追記)
2. ユーザーが端末ごとにアプリ内で読み込み、ブラウザのストレージへ保存する

## 決定

**音声もキューもデバイスローカルに保存する (選択肢 2)。リポジトリには一切コミットしない。**

- MP3 Blob は **IndexedDB** (DB `italyshadowing` / store `audio`、キー = 教材 filename) に保存する — `src/lib/audioStore.js`
- キューは **localStorage** (`italyshadowing.cues.<filename>`、行開始秒の昇順 JSON 配列) に保存する — `src/lib/cueStore.js`
- キューはアプリ内の「キュー記録モード」で作成する: 音声を再生しながら各行の頭でタップ / Space を押し、`audio.currentTime` を記録する
- 教材 JSON のスキーマは変更しない。`scripts/validate-lessons.mjs` も変更不要 (ADR-003 のビルドパイプラインは無変更)

## 理由

- **著作権**: 想定する音源 (語学講座の録音など) は再配布できないものが多く、公開リポジトリ / GitHub Pages に置けない。端末内で完結させれば配布が発生しない。
- **ストレージの選び分け**: MP3 は数 MB 以上のバイナリで localStorage (文字列のみ・容量小) には置けないため IndexedDB を使う。キューは小さな数値配列なので localStorage で十分。
- **キューがデバイスローカルで一貫する**: キューは「その端末に読み込まれた、その音声ファイル」に対する相対時刻であり、端末をまたいで共有しても対応する音声がなければ意味を持たない。音声がローカルである以上、キューもローカルが自然。
- 教材 JSON のインライン取り込み (`import.meta.glob`) の仕組みはバイナリに適用できず、音声には別の読み込み経路がどのみち必要だった。Blob → `URL.createObjectURL` はサブパス配備の `BASE_URL` 問題とも無縁。

## ADR-005 との関係 (永続化失敗の扱い)

ADR-005 の原則どおり、保存・復元はすべて try/catch で包み、失敗してもアプリ本体 (テキスト表示・ページ送り) は動作し続ける。ただし音声の保存は「ユーザーが投入したデータの中核機能」であるため、ADR-005 の例外条項を適用する。

- `putAudioBlob` の失敗は握りつぶさず `false` を返し、UI が「この端末への保存に失敗しました。今回のセッション中のみ再生できます」と通知する。読み込んだ音声はメモリ上 (object URL) で当該セッション中は再生できる。
- `getAudioBlob` / キューの読み込み失敗は「未保存」と同じ扱いに落とす (通知しない)。

## キューの整合性

`loadCues(filename, lineCount)` は次をすべて満たさない保存値を `null` (= 未記録) として扱う:

- 有限数 (`Number.isFinite`) かつ 0 以上
- 狭義単調増加 (昇順・重複なし)
- **配列長が教材の行数と一致**

教材 JSON が後から編集されて行数が変わると長さ検証で弾かれ、自動的に「キュー未記録」状態へ戻る (再生は可能なまま、再記録を促すヒントが表示される)。破損データの部分利用はしない。

## 既知の制約

- 端末ごとに音声の読み込みとキュー記録が必要 (共有されない)
- Safari は ITP により、ホーム画面に追加していない利用で 7 日間操作がないと IndexedDB / localStorage を削除することがある。その場合は音声の再読み込み・キューの再記録が必要。PWA としてホーム画面へ追加した場合は対象外
- ブラウザのプライベートモードでは保存に失敗する (上記の通知付きセッション内フォールバックで動作)
